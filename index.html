Pojeto de bloco TP1

1. Definição de Projeto: A aplicação escolhida para ser desenvolvida foi MindCare.

2. Definição de Escopo e Objetivos: O objetivo da aplicação é conectar psicólogos e pacientes, oferecendo privacidade e devendo oferecer gestão de agendas e registro de sessões para os psicólogos, e para os pacientes deve oferecer acesso facilitado aos psicólogos e acompanhamento do tratamento.

3. Gestão Ágil do Projeto: O trabalho será organizado com transpârencia(compreender o trabalho, progresso e desafios), inspeção(para identificar os problemas cedos) e adaptação(com base na inspeção adaptar para corrigir o rumo e solucionar os problemas). O ciclo do trabalho (sprints) serão de uma semana de acordo com a necessidade de entrega dos tps e definirei as tarefas pelo miro, divididas em pequenas partes. Todo dia será resolvido uma tarefa definida no miro para entregar a tarefa a tempo.

Primeira versão do cronograma do desenvolvimento do projeto:

  1. Criar um design para o site
  2. Criar ambiente de login
  3. Criar interface do site
  4. Criar ambiente de agendamento
  5. Criar ambiente para registro de sessões
  6. Criar ambiente para o acompanhamento das sessões para os pacientes
  7. Criar a diferenciação dos elementos aparentes para os psicólogos e pacientes
  8. Aplicar o site para web

4. Artefatos, Papéis e Eventos do Scrum: Os papéis do scrum (Product Owner, Development Team e Scrum Master) será exercido por eu, Isabela. Irei criar um product backlog para o projeto, e criar uma checklist que será feita de acordo com a sprint backlog.

5. Criação de Histórias de Usuário: Irei criar as histórias do usuário de acordo com o cenário entregue pelo professor.

Como um psicólogo
Quero gestão de agenda, registro de sessões e comunicação protegida
Para que tenha o controle total sobre seu horário de trabalho e formas de contato

Como um paciente
Quero (que) tenha acesso facilitado a terapeutas e acompanhamento do tratamento
Para que tenha possibilidade de encontrar terapeutas com abordagens específicas para suas necessidades individuais

6. Planejamento do Projeto de Desenvolvimento de Front-end: Fazer uma lista dinâmica para desenvolver uma plataforma eficiente e interativa, além de uma interface que seja agradável para os usuários.

7. Interatividade na Aplicação Web/Mobile: Uma barra interativa para acessar diferentes ambientes, busca por terapeutas com filtro de abordagem, interação com a agenda entre outros. Essas interações serão feitas com uma reação clara para entendimento imediato do usuário, seguindo o padrão de design estabelecido, para obter uma ótima experiência do usuário.

8. Framework: Os frameworks ReactJS e React Native são ferramentas que ajudam a criar interações intuitivas e fluídas, sendo capaz de alterar o estado dos componentes ajudando tambem com a vasta biblioteca de componentes e ferramentas.
